# Coding Standard

## Backend

- Keep API routes thin.
- Put business flow in services.
- Put persistence in repositories.
- Keep parser logic separate from sync logic.
- Add focused tests for parser, calculator, repository, and sync behavior.

## Frontend

- Keep feature code under `frontend/src/features`.
- Keep shared API config under `frontend/src/shared`.
- Keep dashboard UI practical and scan-friendly.

## Documentation

- Update `PRD.md` when work changes progress.
- Update root documentation when architecture, API, database, or deployment changes.
